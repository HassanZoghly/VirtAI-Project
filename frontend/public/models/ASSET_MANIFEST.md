## 2. Animation Tracks (FBX Files)

### Track Naming Convention
A byte-level inspection of the FBX animations (`Idle.fbx` and `Talk_1.fbx`) was performed looking for string identifiers surrounding skeletal data (e.g., matching `Spine`, `Hips`). 
- **Findings:** The FBX tracks use the exact clean string names. There are **NO prefixes** (like `mixamorig:` or `Armature|`) attached to the bones in these files.
- **Evidence:** Extracted raw string blocks literally resolve to `'Spine2'`, `'Spine'`, `'Spine1'`, and `'Hips'`. They perfectly mirror the naming of the `avatar1.glb`.

### Retargeting Regex Rule
Because the current FBX tracks directly match the GLB node names without any prefixes, no aggressive renaming is structurally required for *these* specific files. 

However, standard downloaded Mixamo animations typically use `mixamorig:` prefixes. To make `useAvatarAnimations.ts` perfectly robust for both these local clean files AND any future standard Mixamo files, you must use the following standard defensive regex string replacement:

```javascript
// Exact Regex to be used in useAvatarAnimations.ts when mapping animation tracks:
const cleanTrackName = track.name.replace(/mixamorig:|Armature\|/gi, '');
```

*This rule strips both standard Mixamo and Blender root prefixes, yielding the raw `Spine`, `Hips` format identical to our established `avatar1.glb` ground truth.*
